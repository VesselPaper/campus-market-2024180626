## 4. 遇到的问题与解决方法

**问题 1：Edit 工具缩进不一致导致路由文件修改失败**

在删除种子页面路由时，Edit 工具的 old_string 匹配因为 tab 和空格混用而重复失败。反复尝试了不同缩进格式才匹配成功。

**解决方法**：先 Read 文件确认实际缩进方式，再针对性地编写 old_string。后续类似操作应先用 Read 工具确认文件当前状态。

**问题 2：CSS 注释中英文混用**

部分组件之前已有英文注释，另一部分是新加的中文注释，风格不统一。需要逐个文件排查。

**解决方法**：逐一检查所有 12 个 views 和 12 个 components 的 CSS 部分，将残留的英文注释全部替换为中文。同时统一了注释风格：区块注释用 `/* ── 说明 ── */`，行内注释用 `/* 说明 */`。

**问题 3：HomeView 中出现重复的 `const userStore = useUserStore()` 行**

AI 编辑时插入的新代码和原有代码产生了重复定义，导致编译警告。

**解决方法**：Read 文件确认重复位置后，通过 Edit 删除了多余的一行。后续 AI 大范围编辑后应检查有无变量重复定义。

## 6. 今日反思

今天的任务主要聚焦在"体验优化"上，这和之前几天"功能开发"的节奏完全不同。功能开发时，关注的是"这个功能能不能跑通"；体验优化时，关注的是"用户用起来舒不舒服"。

加载状态、空状态、错误状态这三个状态在前面几天其实一直被忽略了。列表数据加载时页面直接空白，搜索无结果时也空白，用户完全不知道发生了什么。今天给列表页、图表、轮播都补上了对应的状态提示，用户体验好了很多。

另一个体会是：不要等全部功能写完再来统一优化。今天的优化过程中，发现有些组件因为之前埋点不够，需要修改多处代码才能补上 loading 状态。如果一开始写功能时就预留好状态变量，后面优化会轻松很多。

代码清理也是一个重要环节。项目经过 AI 多次迭代修改后，产生了不少冗余代码、无用导入和不一致的注释风格。今天花了一些时间做清理和统一，项目代码看起来干净多了。特别是有 4 个种子页面其实早就被 HomeView 的重定向逻辑绕过了，但一直留在项目里没有删除，直到今天才清理掉。
