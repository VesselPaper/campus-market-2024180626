## 5. 遇到的问题与解决方法

**问题：AI 生成的大范围修改可能导致文件损坏**

在 Day6-D7 使用 AI 进行批量编辑时，Edit 工具偶尔会因为缩进不匹配或重复匹配而修改错误位置。

**解决方法**：
1. 每次 AI 大范围修改前先 Git 提交当前稳定版本
2. 修改后立即验证项目是否可正常运行
3. 遇到 Edit 失败时，先 Read 文件确认当前内容再重试
4. 重要修改单独提交，不要和多个改动混在一起

## 6. 实训总结与反思

### 6.1 完成的成果

7 天实训完成后，"校园轻集市"项目实现了以下功能闭环：

1. **本地身份系统** — 注册/登录、localStorage 持久化、会话恢复
2. **首页概览** — 轮播 Banner、快捷入口、热门推荐
3. **集市列表** — 四类信息展示、搜索、筛选、排序
4. **信息详情** — 动态路由、类型专属字段、收藏、砍价
5. **信息发布** — 统一表单、图片上传、类型动态切换
6. **消息中心** — 会话管理、聊天记录、未读红点
7. **个人中心** — 用户资料、我的发布、我的收藏
8. **数据看板** — 4 个 ECharts 统计图表
9. **浏览记录** — localStorage 历史记录、清空
10. **账号设置** — 用户信息查看与管理

### 6.2 技术能力提升

- **Vue3 技能**：Composition API、script setup、组件化开发
- **路由管理**：命名路由、动态路由、导航守卫、路由重定向
- **状态管理**：Pinia Store 设计、跨页面状态同步
- **HTTP 通信**：Axios 封装、参数映射、错误处理
- **Mock API**：JSON Server 数据建模、CRUD 操作
- **UI 框架**：Element Plus 组件使用与样式覆盖
- **数据可视化**：ECharts 图表配置与数据绑定
- **交互优化**：加载状态、空状态、动画反馈

### 6.3 AI 协作体会

通过 7 天实训，对 AI 辅助开发有了更深的理解：

1. **AI 适合生成框架和模板**，不适合处理精细的业务逻辑
2. **AI 生成代码后必须审查**，特别是变量命名、逻辑边界和错误处理
3. **AI 大范围修改前一定要 Git 提交**，防止修改不可恢复
4. **提示词越具体，AI 输出越准确**，模糊的需求会导致偏离预期的代码
5. **AI 不能替代人工理解**，学生必须能够解释每一段关键代码的作用

### 6.4 改进方向

如果继续开发这个项目，可以考虑：
1. 接入真实后端数据库，替代 JSON Server
2. 实现真实用户认证（JWT）
3. 加入实时消息推送（WebSocket）
4. 优化移动端适配体验
5. 添加商品分类管理和后台管理功能
